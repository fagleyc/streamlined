Overview

Streamlined is a PyQt6 wind tunnel data reduction application that replaces the legacy MATLAB workflow. It reads raw force-balance runs, applies multi-order balance and pressure calibrations, transforms loads through the body (BRF) and wind (WRF) reference frames with proper tare subtraction, computes compressible-isentropic tunnel conditions, and non-dimensionalizes to aerodynamic coefficients. Every stage prior to the final steady-state average operates on full time-series vectors — the reported steady value is the temporal mean of fully reduced coefficient time histories, and the per-point standard deviation captures flow unsteadiness.

  • Validated MATLAB-to-Python parity (coefficients within machine precision)
  • Force and moment internal balance configurations, plus external (ATE) balance runs
  • Multi-geometry support with per-case assignment
  • Selectable blockage corrections and stability-derivative plots
  • Modular custom data calculator (ParaView-style expressions)
  • Export to CSV, Excel, HDF5, MAT, and legacy COE

Launch with python streamlined.py from the project root. The technical reference (equations for every stage) is in README.md, available in-app via Help → Technical Reference (README).

Data Formats Read

TDMS files (NI DAQ)

Raw runs from National Instruments DAQ systems, read with nptdms (utils/windtunnel/data_io.py → read_tdms_file()).

[AirState]_[Configuration]_Alpha_[value]_Beta_[value].tdms
AirOn_F16check_Alpha_-2.0_Beta_0.0.tdms
AirOff_F16check_Alpha_0.0_Beta_0.0.tdms
  • All channels from all groups are extracted and resampled to the highest-rate channel's time base via cubic interpolation.
  • Alpha/Beta come from DAQ channels, then TDMS group properties, then filename parsing (in that priority order).
  • Expected channels: N1 N2 Y1 Y2 Axial Roll Excitation Pdiff Ptot Temp Alpha Beta plus optional pressure ports P001, P002, …
  • Files are classified Air-On / Air-Off automatically from the filename prefix.

Freestream / Conductor run files (.h5 / .mat)

Streamlined also reads HDF5 and MATLAB run files written by the freestream suite (read_hdf5_file(), read_mat_file(); routing by extension in read_data_file()). These files are self-describing:

MarkerMeaning
balance_groupWhich group holds balance data: StrainBook_0 (internal sting balance, bridge volts) or ATE_Balance (external balance, resolved loads).
balance_typeinternal (needs .vol calibration via calc_brf_forces) or external (channels already forces/moments).
load_unitsExternal runs carry per-channel unit attributes (freestream writes N / N*m); Streamlined converts to the historical lb / in-lb chain automatically.
span_configfull / half marker on freestream mode 2 files; carried through in file properties (no scaling applied, matching the legacy pipeline).

Filenames of the form …_mach_0.30.h5 carry the Mach set point. If markers are absent, the balance type is inferred from which groups are present.

External (ATE) balance handling

utils/windtunnel/external_balance.py ports the MATLAB external-balance chain (deprecated/scripts/calc_coeffs.m 'External' case):

  • Channel order Drag, Side, Lift, Roll, Pitch, Yaw — already resolved wind-axis loads, so no volts→forces calibration is applied.
  • Per-channel calibration bias values from the historical cal file are subtracted.
  • SI loads (N / N·m from freestream) are converted to lb / in-lb via external_loads_to_ips() before Q·S normalization.
  • Tare uses the same mean-subtraction step as the internal path (subtract_wrf_forces), preserving air-on dynamics.
  • Coefficients divide by Q·S (forces) and Q·S·C (all three moments use reference chord, matching the legacy chain); no MRC transfer is applied to external loads.
  • External-balance bias / precision / total uncertainty is ported in calc_uncertainty_ext_balance().

Calibration files

  • .vol — balance calibration ("Voltage Calibration File 3.1"): balance description, max loads, moment-arm distances (dx1/dx2/dy1/dy2), and positive/negative loading curves for each of the 6 channels. Fit types: Linear (6×6), Quadratic (12×6), Cubic (18×6, default) least-squares matrices. These files are produced by the balance_cal utility in the freestream suite.
  • .PCF — pressure transducer calibration: slope (PSI/V) per transducer (e.g. "220" differential, "690" total), excitation, units.

Reduction Pipeline

Nine stages; stages 1–8 operate on full time-series (see README for the equations):

#StageSourceWhat happens
1Load & resampledata_io.py Read TDMS/h5/mat, resample all channels to a common time base.
2Balance calibrationcalibration.py Parse .vol, zero-offset correct, normalize by excitation, least-squares fit the calibration matrix (R² and RMSE per channel).
3Apply calibrationtransforms.py V/Vex polynomial terms × matrix → element force time-series (skipped for external balance).
4BRF forcestransforms.py Element sums/differences with moment arms and MRC shift → Fx Fy Fz Mx My Mz. Force config (N1/N2/Y1/Y2) and moment config (AftPitch/AftYaw/FwdPitch/FwdYaw) both supported.
5WRF transformtransforms.py Rotate by each dataset's own alpha/beta → Lift Drag Side Roll Pitch Yaw.
6Tare subtractiontransforms.py Subtract the mean of each air-off WRF channel from the air-on time-series (DC removal in WRF, correct at any model attitude).
7Tunnel conditionscoefficients.py Compressible isentropic Q, Mach, Tstatic, rho, a, U∞; Sutherland viscosity; Reynolds number. Thermocouple cal vintage (0.1 V/°C vs 0.1 V/°F) auto-detected per sample.
8Coefficientscoefficients.py CL CD CY over Q·S; Cl/Cn use span, Cm uses MAC. Cp for pressure ports.
9Steady-statereduction.py Temporal mean per point (value) and std (unsteadiness); alpha/beta grid organization.

Downstream of the pipeline: stability derivatives (Cma, CLa, static margin, CYb, Cnb, Clb via central differences), drag-polar fits, Oswald efficiency, and selectable blockage corrections (Pope-Harper, Maskell, Glauert closed-section) from Edit → Tunnel Corrections…

Typical workflow

  1. Load calibrations — Edit → Calibration: pick the balance .vol (choose Linear/Quadratic/Cubic and Force/Moment config) and pressure .PCF. External-balance runs need no .vol.
  2. Define geometry — Edit → Geometry: MAC, span, reference area, MRC offset, input units. Multiple named geometries supported; right-click a case to assign one (re-reduction is automatic).
  3. Load data — File → Load Data: pick one or more run directories; Air-On/Air-Off files are paired automatically.
  4. Process — the controller reduces every case through the 9-stage pipeline; failures are reported with actionable dialogs.
  5. Review — plots, data table, time history / FFT.
  6. Export — File → Export.
  7. Save the configuration as a JSON preset (relative paths) so the whole setup reloads in one step.

Plot view

  • Coefficient plots (CL/CD/Cm/L-D/CY/Cl/Cn vs alpha, drag polar, lateral vs beta, stability derivatives) with multi-case overlay and a "Plot vs β" toggle.
  • ±1σ shading ("Show Std Dev"), alpha/beta multi-select filters, legend/grid toggles, custom-variable Y axis.
  • Right-click a data point: view its time history or FFT.
  • Save Image opens an interactive export dialog: per-trace colors/markers/widths, legend labels, fonts, axis limits, theme (White/Dark/Black/Transparent), resolution — settings persist between sessions.
  • pyqtgraph is the fast interactive backend; matplotlib is the fallback.

Table and Time History views

  • Data Table: all steady-state coefficients with optional tunnel-condition columns; numeric sorting; export via File → Export.
  • Time History: raw and reduced time-series per point with multi-case overlay and FFT; element labels adapt to the balance config (N1/N2/Y1/Y2 or AftPitch/AftYaw/FwdPitch/FwdYaw).

Custom Calculator

Edit → Custom Calculator defines expression rules that derive new variables from any reduced data (e.g. Cp{i} with range auto:p{i} expands to Cp1…Cp32 from one rule). Variables are browsable by category (Pressure Ports, Balance Channels, Tunnel Conditions, BRF/WRF Forces, Coefficients, Geometry, Position/Time, Constants); geometry scalars (MAC, span, ref_area, MRC) are available in expressions. Custom variables flow through plots, the table, exports, and the time-history viewer.

Troubleshooting

SymptomCause / fix
"Invalid Calibration File" dialog The .vol/.PCF is corrupt or the wrong format — re-export from balance_cal or check that a .vol (not .PCF) was selected for the balance.
No files found when loading a directory Runs must be .tdms, .h5, or .mat; TDMS names need the AirOn/AirOff…Alpha…Beta convention. Pre-flight checks list exactly what is missing.
Coefficients look wildly wrong at low speed Q·S floor kicks in at wind-off; check the .PCF slopes and that Pdiff/Ptot channels are present.
Temperatures off by ~20° Thermocouple cal vintage mis-detected on unusual data; force it with temp_cal_mode='degC' or 'degF' on calc_tunnel_conditions() / reduce_raw().
Cma doesn't change after moving the MRC Fixed in v1.2.7 — the Geometry dialog now re-reduces affected cases; update if on an older build.
External-balance file reduces as internal (or vice versa) Check the balance_type / balance_group markers in the run file; files without markers are classified by which groups are present (ATE_Balance vs StrainBook_0).
Old MATLAB scripts can't find export fields v1.2.7 restructured MAT/HDF5 exports by category (case.Tunnel_Conditions.Q etc.) — update field paths.
About dialog raised NameError Fixed in v1.2.8 (changelog text in an f-string contained literal {i} placeholders).
